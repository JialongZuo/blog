Map作为JAVA中一种集合类，用于存储一对key-value的映射，而HashMap又是最常用的Map之一，它为什么要以Hash来命令这种Map呢？

1.Map接口介绍

Map是java中的一种集合类，用于存储key-value应声，我们通过学习Map接口类了解集合类的具体用途。 通过Map接口我们大概可以看出Map提供了键值对的操作。通过Entry这个类型来映射key-value数据类型。

public interface Map<K,V> {
    //定义Map集合的操作方法，put，get，size，remove，putAll等
    V put(K key, V value);
    V get(Object key);
    boolean containsKey(Object key);
    boolean containsValue(Object value);
 
    //Entry接口，Map中的一个内部接口，用于具体存储一对key-value键值对。
    interface Entry<K,V> {
        K getKey();
        V getValue();
        V setValue(V value);	
        boolean equals(Object o);
        int hashCode();
    }
 
    //获取Entry的Set集合
    Set<Map.Entry<K, V>> entrySet();
}

2.AbstractMap介绍

AbstractMap为Map实现提供了一个基础骨架，它能使你花尽可能小的力气去实现一个Map， HashMap就是其的一个子类。

public abstract class AbstractMap<K,V> implements Map<K,V> {
 
    //在AbstractMap中没有实现put方法，留给子类实现，可以看到他抛出了一个UnsupportedOperationException异常
    public V put(K key, V value) {
        throw new UnsupportedOperationException();
    }
 
    /**
     *通过key值获取value，可以看到get方法先获取Map中的Entry的集合Set，然后通过遍历Set来获取符合要求的值
     *其余方法containsKey，containsValue，remove等方法也是通过类似原理实现
     */
    public V get(Object key) {
        Iterator<Entry<K,V>> i = entrySet().iterator();
        if (key==null) {
            while (i.hasNext()) {remove
                Entry<K,V> e = i.next();
                if (e.getKey()==null)
                    return e.getValue();
            }
        } else {
            while (i.hasNext()) {
                Entry<K,V> e = i.next();
                if (key.equals(e.getKey()))
                    return e.getValue();
            }
        }
        return null;
    }
 
    //他的hashCode的值为Map中所有Entry的hash值的总和
    public int hashCode() {
        int h = 0;
        Iterator<Entry<K,V>> i = entrySet().iterator();
        while (i.hasNext())
            h += i.next().hashCode();
        return h;
    }
 
    //同样没有实现entrySet()，留给子类实现
    public abstract Set<Entry<K,V>> entrySet();
 
  //AbstractMap中的的Entry接口实现
  public static class SimpleEntry<K,V>
        implements Entry<K,V>, java.io.Serializable
    {
        private static final long serialVersionUID = -8499721149061103585L;
 
        private final K key;
        private V value;
 
        public K getKey() {
            return key;
        }
 
        public V getValue() {
            return value;
        }
       
        //key和value的hash值按位异或
        public int hashCode() {
            return (key   == null ? 0 :   key.hashCode()) ^
                   (value == null ? 0 : value.hashCode());
        }
    }
}

3.HashMap分析

接下来让我们详细分析一下HashMap，下面介绍它的重要构成。

  • Node-HashMap中的Entry

    Node 为HashMap中的Entry的具体实现，扩展了原有Entry的key-value形式，将其变为了 一种单向链表。

    /**
     *和AbstractMap中的SimpleEntry相比多出来hash和next两个属性，
     *next类似单向链表一样将HashMap中的Node串联起来。
     */
    static class Node<K,V> implements Map.Entry<K,V> {
        final int hash;
        final K key;
        V value;
        Node<K,V> next;
 
        Node(int hash, K key, V value, Node<K,V> next) {
            this.hash = hash;
            this.key = key;
            this.value = value;
            this.next = next;
        }
 
        public final K getKey()        { return key; }
        public final V getValue()      { return value; }
        public final String toString() { return key + "=" + value; }
 
        public final int hashCode() {
            return Objects.hashCode(key) ^ Objects.hashCode(value);
        }
 
        public final V setValue(V newValue) {
            V oldValue = value;
            value = newValue;
            return oldValue;
        }
 
        public final boolean equals(Object o) {
            if (o == this)
                return true;
            if (o instanceof Map.Entry) {
                Map.Entry<?,?> e = (Map.Entry<?,?>)o;
                if (Objects.equals(key, e.getKey()) &&
                    Objects.equals(value, e.getValue()))
                    return true;
            }
            return false;
        }
    }

  • HashMap中的主要属性介绍

    /**
     * 当第一次使用时初始化和在需要时重新调整大小，数组大小总是2的词幂
     * 类似一个<upper>分类箱</upper>先通过key通过hash方法得到一个hash然后再通过(table.length-1)& hash进行分类放入对应的数组空间中.
     * 可以发现table存储类型相同（通过如上方法计算值相同）的Node，然后table每一个位置存储一个Node（单向链表,类型相同的所有Node链起来的链表）.
     * 它被声明称一个tarnsient字段，因为其实现了writeObject来进行自定义序列化，在其中处理这这个字段。其他字段同理。
     * The table, initialized on first use, and resized as
     * necessary. When allocated, length is always a power of two.
     * (We also tolerate length zero in some operations to allow
     * bootstrapping mechanics that are currently not needed.)
     */
    transient Node<K,V>[] table;
 
    /**
     * 缓存entrySet方法的返回值
     * Holds cached entrySet(). Note that AbstractMap fields are used
     * for keySet() and values().
     */
    transient Set<Map.Entry<K,V>> entrySet;
 
    /**
     * Map中键值对的总数
     * The number of key-value mappings contained in this map.
     */
    transient int size;
 
    /**
     * Map修改次数计数器
     * The number of times this HashMap has been structurally modified
     * Structural modifications are those that change the number of mappings in
     * the HashMap or otherwise modify its internal structure (e.g.,
     * rehash).  This field is used to make iterators on Collection-views of
     * the HashMap fail-fast.  (See ConcurrentModificationException).
     */
    transient int modCount;
 
    /**
     *当size大于capacity*factor时，会执行resize操作
     * The next size value at which to resize (capacity * load factor).
     *
     * @serial
     */
    // (The javadoc description is true upon serialization.
    // Additionally, if the table array has not been allocated, this
    // field holds the initial array capacity, or zero signifying
    // DEFAULT_INITIAL_CAPACITY.)
    int threshold;
 
    /**
     * 装载因子,用于衡量HashMap满的程度，默认默认为0.75f
     * The load factor for the hash table.
     *
     * @serial
     */
    final float loadFactor;

  • HashMap中的put方法

    让我们通过put方法来分析HashMap中的数据存在形式

    //存储key-value到Map中
    public V put(K key, V value) {
        return putVal(hash(key), key, value, false, true);
    }
 
    //计算key的hash值，方法为key的hashCode与其无符号右移16位的异或值
    static final int hash(Object key) {
        int h;
        return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
    }
    
    /*
     * 核心方法通过这个方法可以看出HashMap的具体实现原理和思路，让我们来具体分析一下。大体思路就是先通过key和算出hash然后根据
     * 一系列的算式得到Node在table中的具体位置，如果位置当前已经有了Node则将Node遍历判断是否已经存在，如果不存在则放入链表尾部。
     */
    final V putVal(int hash, K key, V value, boolean onlyIfAbsent,
                   boolean evict) {
        Node<K,V>[] tab; Node<K,V> p; int n, i;
        if ((tab = table) == null || (n = tab.length) == 0)
            //resize方法，初始化table或者当容量等于门槛值(threshold)的时候扩容。下面会详细讲解这个方法。
            n = (tab = resize()).length;
        //如果所需放入table数组的当前位置没有值，则直接插入此位置。（所需位置用过hash值来确定，
        //可以说hash值确定了Node的类型将Node进行分类。由此可以看出HashMap是一种无序的Map。
        if ((p = tab[i = (n - 1) & hash]) == null)
            tab[i] = newNode(hash, key, value, null);
        else {
            Node<K,V> e; K k;
        //如果key值相同，则e为新的value（e为需要插入的新value，在随后通过onlyIfAbsent判断策略来决定是否插入）
            if (p.hash == hash &&
                ((k = p.key) == key || (key != null && key.equals(k))))
                e = p;
            else if (p instanceof TreeNode) 
                e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
            else { //如果当前位置已经有值，遍历node链表如果没有key相同的则将其添加到链表尾部。
                for (int binCount = 0; ; ++binCount) {
                    if ((e = p.next) == null) {
                        p.next = newNode(hash, key, value, null);
                        if (binCount >= TREEIFY_THRESHOLD - 1) // -1 for 1st
                            treeifyBin(tab, hash);
                        break;
                    }
                    if (e.hash == hash &&
                        ((k = e.key) == key || (key != null && key.equals(k))))
                        break;
                    p = e;
                }
            }
            if (e != null) { // existing mapping for key
                V oldValue = e.value;
                if (!onlyIfAbsent || oldValue == null)
                    e.value = value;
                afterNodeAccess(e);
                return oldValue;
            }
        }
        ++modCount;
        if (++size > threshold)
            resize();
        afterNodeInsertion(evict);
        return null;
    }

  • MashMap中的resize()

    resize方法表现了HashMap具体的初始化和扩容策略

    //这个方法就是初始化table[]和当容量（size）超过临界值（threshold）时调整大小。Map初始化和map中size
    //到达容量临界值（threshold）时调用此方法重新分配合适大小的table
    final Node<K,V>[] resize() {
        Node<K,V>[] oldTab = table;
        int oldCap = (oldTab == null) ? 0 : oldTab.length;
        int oldThr = threshold;
        int newCap, newThr = 0;
        if (oldCap > 0) {//旧Node数组有值，即调用此方法来扩容数组则新的扩容临界值设置为原来的2倍。
            if (oldCap >= MAXIMUM_CAPACITY) {
                threshold = Integer.MAX_VALUE;
                return oldTab;
            }
            else if ((newCap = oldCap << 1) < MAXIMUM_CAPACITY &&
                     oldCap >= DEFAULT_INITIAL_CAPACITY)
                newThr = oldThr << 1; // double threshold
        }
        else if (oldThr > 0) // initial capacity was placed in threshold。初始化容量，如果有临界值则将容量设置为临界值。
            newCap = oldThr;
        else {               // zero initial threshold signifies using defaults
            //初始化容量，如果扩容临界值为0，算出新的容量为默认值16,新的扩容临界值为16×0.76（默认）
            newCap = DEFAULT_INITIAL_CAPACITY;
            newThr = (int)(DEFAULT_LOAD_FACTOR * DEFAULT_INITIAL_CAPACITY);
        }
        if (newThr == 0) {
            float ft = (float)newCap * loadFactor;
            newThr = (newCap < MAXIMUM_CAPACITY && ft < (float)MAXIMUM_CAPACITY ?
                      (int)ft : Integer.MAX_VALUE);
        }
        threshold = newThr;
        @SuppressWarnings({"rawtypes","unchecked"})
            Node<K,V>[] newTab = (Node<K,V>[])new Node[newCap];//创建一个容量为（newCap)的Node数组赋值给table
        table = newTab;
        if (oldTab != null) { //如果老Node数组有值（即调用此方法为table扩容而不是初始化）则拷贝原数组数据到新的数组中。
            for (int j = 0; j < oldCap; ++j) {
                Node<K,V> e;
                if ((e = oldTab[j]) != null) {
                    oldTab[j] = null;
                    if (e.next == null)
                        newTab[e.hash & (newCap - 1)] = e;
                    else if (e instanceof TreeNode)
                        ((TreeNode<K,V>)e).split(this, newTab, j, oldCap);
                    else { // preserve order
                        Node<K,V> loHead = null, loTail = null;
                        Node<K,V> hiHead = null, hiTail = null;
                        Node<K,V> next;
                        do {
                            next = e.next;
                            if ((e.hash & oldCap) == 0) {
                                if (loTail == null)
                                    loHead = e;
                                else
                                    loTail.next = e;
                                loTail = e;
                            }
                            else {
                                if (hiTail == null)
                                    hiHead = e;
                                else
                                    hiTail.next = e;
                                hiTail = e;
                            }
                        } while ((e = next) != null);
                        if (loTail != null) {
                            loTail.next = null;
                            newTab[j] = loHead;
                        }
                        if (hiTail != null) {
                            hiTail.next = null;
                            newTab[j + oldCap] = hiHead;
                        }
                    }
                }
            }
        }
        return newTab;
    }

  • HashMap中的get方法

    回想HashMap的put方法，那么get方法就很好理解了。

    public V get(Object key) {
        Node<K,V> e;
        return (e = getNode(hash(key), key)) == null ? null : e.value;
    }

    /**
     * Implements Map.get and related methods
     * 通过get方法可以看出同put方法思路一样，先通过key算出所在分类箱table的位置然后遍历链表返回key所对应的值。
     * @param hash hash for key
     * @param key the key
     * @return the node, or null if none
     */
    final Node<K,V> getNode(int hash, Object key) {
        Node<K,V>[] tab; Node<K,V> first, e; int n; K k;
        if ((tab = table) != null && (n = tab.length) > 0 &&
            (first = tab[(n - 1) & hash]) != null) {
            if (first.hash == hash && // always check first node
                ((k = first.key) == key || (key != null && key.equals(k))))
                return first;
            if ((e = first.next) != null) {
                if (first instanceof TreeNode)
                    return ((TreeNode<K,V>)first).getTreeNode(hash, key);
                do {
                    if (e.hash == hash &&
                        ((k = e.key) == key || (key != null && key.equals(k))))
                        return e;
                } while ((e = e.next) != null);
            }
        }
        return null;
    }
